The Challenge

High IBR Penetration Is Breaking Legacy Operator Tools

⚡ Faster, More Complex Dynamics

Inverter-based resources (IBRs) respond in milliseconds — 10–100× faster than synchronous machines. Grid events now unfold on sub-second timescales that legacy stability tools were never designed to handle.

📊 SCADA Data Overload

Operators face vast real-time measurement streams with no integrated, forward-looking tool to anticipate contingency consequences within the tight time windows imposed by IBR-driven grid dynamics.

🔍 DSA Tools Ill-Suited for IBR Grids

Commercial DSA platforms were built for synchronous-machine paradigms. They cannot capture IBR control interactions, or evolving topology without expensive manual recalibration.

🤖 No Explainable AI in the Control Room

Existing AI security classifiers lack natural-language rationale and audit trails — making them incompatible with operator trust, override authority, and EU AI Act accountability requirements.

Architecture

Three Pillars of the GenDT-Ops Framework

🌞 Generative State Model

Score-based diffusion model conditioned on live SCADA and NWP forecasts samples diverse near-future operating points capturing IBR-dominated grid dynamics. Physics consistency layer ensures all generated trajectories are AC power-flow feasible. Generated states seed the Digital Twin for N-k DSA.

🔗 On-Demand Digital Twin

Dynawo-based DT instances instantiated on demand — not always running. Each snapshot spawns parallel faster-than-real-time simulations. Tracks frequency nadir, RoCoF, voltage dips, and stability margins across a critical contingency catalogue.

💬 LLM Operator Assistant

Open-weight LLM (LLaMA 3.x / Mistral) fine-tuned on TSO operational procedures and ENTSO-E guidelines. Chain-of-Drafts (CoD) reasoning produces step-by-step, auditable rationales. Accepts natural-language queries and fuses them with live DT predictions.

All components communicate via frozen API contracts — microservice architecture enables modular deployment and integration with existing TSO/DSO EMS/SCADA infrastructure.

Under the Hood

Four AI Pillars

🌞 Diffusion-Based Grid State Model

  • Graph Transformer denoising score network on topology-aware grid representation
  • Conditioned jointly on live SCADA observations and NWP ensemble forecasts
  • Two-stage physics consistency: training-time AC penalty + NR projection at inference
  • Deep Ensembles for epistemic UQ; ECE-calibrated uncertainty estimates

⚛ Faster-Than-Real-Time Digital Twin

  • On-demand Dynawo instances from diffusion-sampled operating points
  • Parallel HPC dispatch for what-if and N-k DSA across all stability domains
  • Latency budget: SCADA <1 s → diffusion <5 s → DT <10 s → LLM <3 s → total <30 s

💬 LLM Operator Assistant

  • LLaMA 3.x / Mistral fine-tuned on TSO procedures & ENTSO-E guidelines
  • Chain-of-Drafts (CoD) auditable reasoning with mandatory numerical cross-checks
  • Layered fallback: schema enforcement → DT-only mode on cross-check failure

🤖 RL Safety Layer

  • Advisory RL agent proposes re-dispatch, switching, and storage actions
  • Two-tier oracle: fast GNN surrogate screens, full Dynawo verifies (~100× fewer calls)
  • Pareto-front training yields operator-selectable priority profiles at deployment

Scientific Novelty

Four Frontier Contributions

1

Diffusion + GNN + NWP for Low-Inertia Grids

Score-based diffusion conditioned jointly on live SCADA and NWP ensembles, with topology-aware GNN denoising and medoid-selected Dynawo training data. Concurrent work addresses physics-constrained generation and LLM grid control separately; GenDT-Ops integrates these capabilities end-to-end.

2

Auditable Chain-of-Drafts for Power System DSA

Schema-constrained CoD reasoning with mandatory numerical cross-checks against live DT/SCADA values. Structured auditability mechanism — not guaranteed faithful explanation — with layered fallback to DT-only mode. First application of schema-constrained CoD to multi-domain DSA.

3

Advisory RL with Verifiable Physics Oracle

RL agent trained over the diffusion-sampled state distribution — not just the current snapshot — with a two-tier oracle (GNN surrogate + Dynawo). Positioned as advisory, not autonomous, consistent with the field’s current proof-of-concept stage.

4

On-Demand DT Orchestration with Generative State Input

DT instances created from diffusion-model-sampled operating points enable probabilistic DSA: “what fraction of the near-future state distribution is secure?” — a qualitative extension over persistent-mirror DT frameworks.

Safety & Compliance

Built for Trust, Audit, and Regulation

🛡 Failure Modes & Fallback

Mandatory numerical cross-checks on every CoD step against DT/SCADA values. Schema enforcement constrains output structure. Automatic fallback to DT-only mode if cross-checks fail — operators always see physics-based indicators.

🌐 EU AI Act — High-Risk

Classified under Annex III, Section 2 (critical infrastructure). Full audit trail: CoD traces, DT I/O, SCADA snapshots, NWP inputs, RL proposals, and operator responses logged in queryable format. Human override authority preserved at every stage.

🔒 NIS2 & Cybersecurity

Attack surface addressed: adversarial SCADA inputs (EMS integrity), prompt injection (OKB sanitisation), API security (LLM–DT interface). Formal threat model and mitigation architecture developed in Phase 2 with TSO partner, per NIS2 Directive obligations.

Design principle: No AI-generated recommendation reaches the operator without passing a physics-based verification gate. The system degrades gracefully — never silently.
